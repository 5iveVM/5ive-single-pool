# 5ive Single-Pool E2E Test Suite

## Overview

Complete end-to-end test suite for validating the Five DSL port of SPL single-pool on a real localnet validator.

## Files Created

| File | Purpose |
|------|---------|
| `e2e-setup.mjs` | One-time setup: initializes pool, derives PDAs, writes `test-env.json` |
| `e2e-single-pool-test.mjs` | Main E2E test: verifies pool lifecycle and operations |
| `run-e2e.sh` | Orchestrator: builds, deploys, runs setup + tests |
| `test-env.json` | Generated by setup; contains all derived addresses |

## Quick Start

### Prerequisites

```bash
# 1. Start localnet
solana-test-validator --reset

# 2. In another terminal, set environment
export FIVE_NETWORK=localnet
export FIVE_RPC_URL=http://127.0.0.1:8899
export FIVE_KEYPAIR_PATH=$HOME/.config/solana/id.json
export FIVE_PROGRAM_ID=5ive5hbC3aRsvq37MP5m4sHtTSFxT4Cq1smS4ddyWJ6h

# 3. Navigate to project
cd /Users/ivmidable/Development/five-mono/5ive-single-pool
```

### Run Full E2E Suite

```bash
# Option 1: Use orchestrator (recommended)
bash run-e2e.sh

# Option 2: Manual steps
five build
five deploy build/main.five  # Note the script account returned
export FIVE_SINGLE_POOL_SCRIPT_ACCOUNT=<script_account>
node e2e-setup.mjs
node e2e-single-pool-test.mjs
```

## Test Flow

### Setup Phase (`e2e-setup.mjs`)

1. **Payer funding** — Ensures payer has at least 5 SOL
2. **Script account** — Uses `FIVE_SINGLE_POOL_SCRIPT_ACCOUNT` env var (or discovers from deployment)
3. **Vote account discovery** — Finds default vote account from `solana-test-validator`
4. **PDA derivation** — Derives all pool PDAs (pool, stake, mint, authorities)
5. **Pool stake funding** — Creates/funds stake account with rent-exempt + 1 SOL
6. **Pool initialization** — Calls `initialize_pool` via FiveProgram
7. **Writes `test-env.json`** — All addresses for test phase

### Test Phase (`e2e-single-pool-test.mjs`)

1. **Verify pool state** — Reads pool account, confirms `account_type == 1`
2. **Create user stake** — Generates keypairs, funds, creates stake account
3. **Test reactivate_pool_stake** — Verifies pool can be re-delegated
4. **Reports CU** — Extracts compute units from transaction logs

## Key Features

- **Embedded ABI** — No dependency on build artifacts (handles empty `abi.functions` in `.five` file)
- **Fee tail injection** — Automatically adds `[payer, fee_vault, system_program]` if missing
- **Error extraction** — Maps Solana/VM errors to readable messages
- **CU tracking** — Extracts and reports compute units for each operation
- **Test isolation** — Each run generates fresh keypairs; no state pollution

## Environment Variables

```bash
# Required
FIVE_NETWORK              # localnet, devnet, or mainnet (default: localnet)
FIVE_RPC_URL              # RPC endpoint (default: http://127.0.0.1:8899)
FIVE_KEYPAIR_PATH         # Payer keypair path (default: ~/.config/solana/id.json)
FIVE_PROGRAM_ID           # Five VM program (default: localnet ID)

# Optional (auto-derived if omitted)
FIVE_SINGLE_POOL_SCRIPT_ACCOUNT   # Script account pubkey
VM_STATE_PDA                      # VM state PDA (derived from program ID)
FEE_VAULT_ACCOUNT                 # Fee vault PDA (derived from program ID)
```

## Troubleshooting

### `localnet not running`
```bash
# Start validator
solana-test-validator
```

### `Could not extract script account from deployment output`
```bash
# Deploy manually and set env var
five deploy build/main.five
# Copy returned script account
export FIVE_SINGLE_POOL_SCRIPT_ACCOUNT=<pubkey>
node e2e-setup.mjs
```

### `Pool account not found on-chain`
- Run `node e2e-setup.mjs` first to initialize
- Or ensure `test-env.json` exists with valid `poolPda`

### `No active vote accounts found`
- `solana-test-validator` starts with a default vote account
- Verify validator is running and fully initialized
